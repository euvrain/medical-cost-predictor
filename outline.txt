WEEK 1: Setup & Exploratory Data Analysis
Day 1–2: Project Setup
Create a GitHub repo with three folders: /data, /notebooks, and /app. Set up a virtual environment and install your core libraries: pandas, numpy, matplotlib, seaborn, scikit-learn, streamlit, and joblib. Download insurance.csv from Kaggle and load it up. Run df.info(), df.describe(), and df.isnull().sum(). Good news: this dataset has zero missing values, so Day 2 is pure exploration.
Day 3–5: Exploratory Data Analysis
This is where you build intuition before touching a model. Make these plots in a Jupyter notebook:

Histogram of charges — you'll see a strong right skew. Then log-transform it (np.log(charges)) and notice how much more normal it looks. This tells you to model log(charges), not charges directly.
Scatter plots of charges vs age, BMI, and number of children.
Box plot of charges grouped by smoker status. This will be your biggest "aha" moment — smokers cost roughly 4x more. This single variable drives most of the model's predictive power.
Correlation heatmap of all numeric features.
Bar chart of mean charges by region (you'll find it's not very different — region is a weak feature).

Document your findings in markdown cells. The smoking insight is your best talking point for this project.

WEEK 2: Feature Engineering & Baseline Model
Day 1–3: Prepare Your Features
This is where your statistical learning coursework kicks in directly.
Encode the categorical columns (sex, smoker, region) using pd.get_dummies(..., drop_first=True). Then create two new engineered features:

age_sq = age ** 2 — captures the nonlinear relationship between age and cost (costs accelerate as you get older, they don't just increase linearly)
bmi_smoker = bmi * smoker_yes — an interaction term. EDA shows BMI barely matters for non-smokers but matters a lot for smokers. This one feature will bump your R² by ~4–5 points.

Set your target as y = np.log(df['charges']). Do an 80/20 train/test split with random_state=42. Then fit a StandardScaler on the training set only, and use it to transform both train and test. Fitting the scaler on test data too is one of the most common beginner leakage mistakes — avoid it.
Day 4–5: Baseline OLS Model
Fit a plain LinearRegression and record your metrics: R², RMSE, and MAE. Because your target is log-transformed, convert predictions back with np.exp() before computing dollar-denominated MAE. You should see R² around 0.84–0.86 and MAE around $2,500–$3,000. That's a solid baseline.
Make two diagnostic plots: predicted vs actual (should cluster around a diagonal line), and a residual plot (residuals vs fitted values). If you see a funnel shape in the residuals, that's heteroskedasticity — the log transformation should mostly fix this, which is exactly why you did it.

WEEK 3: Model Progression
This is the core of your statistical learning and ML coursework showing up in practice. Build all four models and keep a running comparison table.
Day 1–2: Ridge & Lasso
Fit RidgeCV and LassoCV from scikit-learn, both with 5-fold cross-validation and a range of alphas from np.logspace(-3, 3, 100). These pick the best regularization strength automatically.
After fitting Lasso, print the non-zero coefficients. You'll likely find it zeroes out sex and some region dummies — confirming your EDA finding that those are weak features. This is automatic feature selection, and it's a powerful result to discuss. Ridge will keep all features but shrink the weak ones toward zero.
Both should improve slightly on your OLS baseline, landing around R² = 0.85–0.87.
Day 3–5: Random Forest & Final Model Selection
Fit a RandomForestRegressor and tune it with GridSearchCV over n_estimators (100, 300), max_depth (None, 5, 10), and min_samples_leaf (1, 5). This will take a few minutes to run. Your best RF model should hit R² around 0.88–0.90 — your best result.
Extract and plot feature importances as a horizontal bar chart. smoker_yes and bmi_smoker will dominate. This is a great visual for your app and your portfolio.
Pick Random Forest as your final model (best accuracy) but keep Ridge or Lasso as your "interpretable" model for explaining coefficients. Save both with joblib.dump(), along with your scaler and feature column list. You'll need all three in the app.

WEEK 4: Streamlit App & Deployment
Day 1–3: Build the App
Create app/app.py. At the top, load your saved model, scaler, and feature list with joblib.load().
The UI should have: an age slider (18–64), a BMI slider (15–55), a children dropdown (0–5), a smoker toggle, a sex radio button, and a region dropdown. When the user fills these in, you reconstruct the input row exactly as your training data was structured — same column order, same interaction terms (bmi_smoker, age_sq), same encoding. Scale it with your loaded scaler, predict, and np.exp() the result back to dollars.
Display the prediction with st.metric(). Then add one chart: a bar chart of feature importances from your Random Forest, highlighting which factors matter most. A bonus feature that takes 20 minutes and looks impressive: add a "What if I quit smoking?" toggle that shows the predicted cost difference side by side.
The single most common bug here is a mismatch between the features your model was trained on and the features you pass at prediction time. Save your feature column list as a Python list with joblib.dump(list(X.columns), 'features.pkl') and always reorder your input row to match it before predicting.
Day 4–5: Deploy
Create a requirements.txt with pinned versions of all your libraries. Push everything to GitHub — including model.pkl, scaler.pkl, and features.pkl. Go to share.streamlit.io, connect your GitHub repo, set the main file to app/app.py, and click Deploy. It takes about 3 minutes. Your app will have a live public URL you can share anywhere.
